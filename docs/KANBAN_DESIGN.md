# Kanban Task Board — Implementation Design Reference

_Generated by the kanban-design workflow (5 lenses + synthesis). Working reference; not user-facing._

## Synthesis summary

Synthesis verdict: ship a per-conversation kanban board persisted to a NEW `task_boards.json` (Rust), exposed via a single `manage_tasks` tool that overwrites the whole board (TodoWrite-style), driven by a hardened TS agent loop (multi-task heuristic, dynamic MAX_ROUNDS, board re-injection, evidence gate, widened narration regex), surfaced through a calm collapsible kanban strip in ChatPanel, and instructed via one ~22-line appended block in `default_system_prompt`. The kanban is the structural fix for the TEDC failure mode: the model can no longer "queue a plan" without producing a real, persisted, evidence-bearing board, and the loop refuses to let it claim cards done without a co-occurring non-board tool call this round.\n\nThe five lenses overwhelmingly agreed on shape; we resolved (a) where the board lives (separate `task_boards.json`, NOT inside Conversation, to avoid history.rs migration risk), (b) the operation contract (single `set`/whole-board-overwrite, NOT action verbs like create/update — simpler for local models and matches Claude Code's TodoWrite), (c) where validation runs (TS side in `executeTool`, with Rust as a second guard on the Done-evidence-non-empty invariant), (d) UI placement (collapsible strip between `.messages` and `.queue-strip`, horizontal scroll-snap 4-column expanded view, CSS-only animations), and (e) prompt insertion point (between line 144 CRITICAL and line 145 "Ground every claim"). One non-obvious resolution: the `manage_tasks` schema uses STRING ids (Rust assigns UUIDs when client passes empty), not integers — this lets the prompt's worked example carry stable references and matches `Card.id: string` in the UI types.

### [1] NEW `src-tauri/src/task_board.rs`

Per-conversation kanban store. Defines TaskStatus enum (snake_case: todo/in_progress/done/blocked), Task, TaskInput, Board, TaskBoardStore (versioned), BoardSummary. Exposes 4 Tauri commands: get_task_board, set_task_board, list_task_boards, clear_task_board. Owns a `static FILE_LOCK: Mutex<()>` to serialize concurrent agent/UI writes. Atomic write via `<path>.tmp` + rename. Server is sole source of timestamps and attempt_count; bumps attempt_count on todo->in_progress transitions. Validates: Done requires non-empty evidence; Blocked requires non-empty blocker. Returns Err with model-readable message on violation so the agent loop can self-correct.

```rust
use std::sync::Mutex;
use tauri::{AppHandle, Manager};

#[derive(Serialize, Deserialize, Clone, Copy, Debug, PartialEq, Eq, Default)]
#[serde(rename_all = "snake_case")]

#[derive(Serialize, Deserialize, Clone, Debug)]
pub struct Task {
    pub id: String,
    pub title: String,
    pub status: TaskStatus,

    #[serde(default, skip_serializing_if = "Option::is_none")]

    pub updated_at: String,
    #[serde(default)]
    pub attempt_count: u32,
}

#[derive(Deserialize, Clone, Debug)]
pub struct TaskInput {
    #[serde(default)] pub id: String,            // empty => server assigns UUID
    pub title: String,
    pub status: TaskStatus,
    #[serde(default)] pub evidence: Option<String>,
    #[serde(default)] pub blocker: Option<String>,
}

#[derive(Serialize, Deserialize, Clone, Debug, Default)]
pub struct Board {
    pub conversation_id: String,
    pub updated_at: String,
    pub tasks: Vec<Task>,
    #[serde(default)] pub version: u64,         // monotonically bumped per save
}

#[derive(Serialize, Deserialize, Debug, Default)]
pub struct TaskBoardStore {
    #[serde(default = "default_version")] pub schema_version: u32,
    #[serde(default)] pub boards: Vec<Board>,
}
fn default_version() -> u32 { CURRENT_SCHEMA_VERSION }

#[tauri::command]
pub fn set_task_board(app: AppHandle, conversation_id: String, tasks: Vec<TaskInput>) -> Result<Board, String> {
    let _g = FILE_LOCK.lock().unwrap();
    // 1. Evidence guard (Done) + blocker guard (Blocked)
    for t in &tasks {
        if matches!(t.status, TaskStatus::Done) && t.evidence.as_deref().map(str::trim).unwrap_or("").is_empty() {
            return Err(format!("Task '{}' marked done without evidence. Supply an evidence string referencing the tool call(s).", if t.id.is_empty() { &t.title } else { &t.id }));
        }
        if matches!(t.status, TaskStatus::Blocked) && t.blocker.as_deref().map(str::trim).unwrap_or("").is_empty() {
            return Err(format!("Task '{}' marked blocked without a 'blocker' reason.", if t.id.is_empty() { &t.title } else { &t.id }));
        }
    }
    // 2. Diff vs existing board, preserve created_at, bump attempt_count on todo->in_progress (or blocked->in_progress).
    // 3. Assign UUID where id is empty. 4. Enforce at most one in_progress (collapse extras to todo, first wins).
    // 5. Bump board.version. 6. Atomic write.
    // (full body in implementation)
    unimplemented!()
}
```

```ts
import type { TaskBoard, TaskCard } from "./types";

export const NARRATION_PATTERNS: { name: string; re: RegExp }[] = [
  { name: "queued_tasks", re: /\bi(?:['’]?ve| have)\s+(?:queued|added|created|noted|listed|broken\s+(?:down|out))\s+(?:all\s+)?(?:the\s+)?(?:tasks?|items?|steps?)\b/i },
  { name: "heres_the_breakdown", re: /\b(?:here(?:['’]?s| is)|below is)\s+(?:the|my|a)\s+(?:breakdown|plan|outline|approach|sequence|task\s+list)\b/i },
  { name: "ill_do_next", re: /\bi(?:['’]?ll| will)\s+(?:do|tackle|handle|address|work\s+on|start\s+with|begin\s+with)\s+(?:this|that|these|next|first|the\s+(?:first|next))\b/i },
  { name: "next_steps_label", re: /(?:^|\n)\s*(?:next\s+steps?|plan|here(?:['’]?s| is)\s+what\s+i(?:['’]?ll| will)\s+do)\s*:/im },
  // keep existing ACTION_CLAIM coverage:
  { name: "i_will_action", re: /\bi(?:['’]?ll| will)\s+(?:search|look\s+up|find|locate|open|check|attempt|handle|retrieve|pull\s+up|fetch|get|analyze|review|read|write|draft|send|schedule|create|update|generate|compile|summarize|extract|run|start|begin|proceed)\b/i },
  { name: "let_me_action", re: /\blet\s+me\s+(?:search|look\s+up|find|locate|open|check|retrieve|pull\s+up|fetch|get|access|analyze|review|read|write|draft|send|schedule|create|run|start|begin|see|investigate|examine|verify|gather)\b/i },
  { name: "ive_done_action", re: /\bi(?:['’]?ve| have)\s+(?:opened|found|located|searched|scheduled|sent|retrieved|drafted|written|analyzed|reviewed|read|created|updated|generated|fetched|pulled|completed|finished)\b/i },
  { name: "searching_for", re: /\b(?:searching|looking|hunting)\s+for\b/i },
];
export const NUMBERED_PLAN = /(?:^|\n)\s*1[.)]\s+\S.+(?:\n|.){1,400}?(?:^|\n)\s*2[.)]\s+\S/m;

export function detectNarration(content: string): string | null {
  for (const p of NARRATION_PATTERNS) if (p.re.test(content)) return p.name;
  if (content.length > 80 && NUMBERED_PLAN.test(content) && !/\?[^?]{0,80}$/.test(content)) {
    return "numbered_plan_no_tool";
  }
  return null;
}

const IMPERATIVE_VERBS = ["search","find","look","check","fetch","get","retrieve","pull","open","read","write","draft","send","email","reply","schedule","book","create","add","update","delete","summarize","analyze","review","compile","extract","list","show","explain","post","upload","download","run","compose","prepare","build","generate","plan","rewrite"];
export function estimateTaskCount(userMsg: string): number {
  const msg = userMsg.trim(); if (!msg) return 0;
  const numbered = msg.match(/(?:^|\n)\s*\d+[.)]\s+\S/g);
  if (numbered && numbered.length >= 2) return numbered.length;
  const lower = msg.toLowerCase();
  const hits = new Set<string>();
  for (const v of IMPERATIVE_VERBS) {
    const re = new RegExp(`(?:^|[.!?]\\s+|\\sand\\s+|\\sthen\\s+|,\\s+)${v}\\b`, "i");
    if (re.test(lower)) hits.add(v);
  }
  if (hits.size >= 2) return hits.size;
  const conn = msg.split(/\s(?:and\s+then|then|after\s+that|also|;\s)\s/i).filter(s => s.trim().length > 4);
  if (conn.length >= 2) return conn.length;
  return 1;
}
export function isMultiTask(userMsg: string): boolean { return estimateTaskCount(userMsg) >= 2; }

export function validateEvidence(evidence: string | undefined, toolsThisRound: Set<string>): { ok: boolean; reason?: string } {
  if (!evidence || evidence.trim().length < 8) return { ok: false, reason: "evidence is empty or too short (<8 chars)" };
  const lower = evidence.toLowerCase();
  for (const t of toolsThisRound) if (t !== "manage_tasks" && lower.includes(t.toLowerCase())) return { ok: true };
  if (/(?:\/Users\/|\/Library\/|~\/)/.test(evidence)) return { ok: true };
  if (/\bhttps?:\/\//i.test(evidence)) return { ok: true };
  if (/(?:`[^`]+`|\b(?:msg_|event_|conv_|file_|task_)[\w-]{4,})/.test(evidence)) return { ok: true };
  if (/\b\d{2,}\s*(?:bytes|chars|lines|results|matches|rows|kb|mb|messages|emails|events|files|slides|hits)\b/i.test(evidence)) return { ok: true };
  return { ok: false, reason: "evidence must name a tool that ran this round, a path, a URL, a quoted ID, or a number-with-unit" };
}

export function renderBoardSnapshot(b: TaskBoard): string {
  const cols: Record<string, TaskCard[]> = { in_progress:[], todo:[], blocked:[], done:[] };
  for (const c of b.tasks) cols[c.status]?.push(c);
  const line = (c: TaskCard) => `  - [${c.id.slice(0,6)}] ${c.title}${c.attempt_count>1?` (×${c.attempt_count})`:""}${c.evidence?` ev: ${c.evidence.slice(0,80)}`:""}${c.blocker?` blocker: ${c.blocker}`:""}`;
  return [`IN_PROGRESS (${cols.in_progress.length}):`, ...cols.in_progress.map(line), `TODO (${cols.todo.length}):`, ...cols.todo.map(line), `BLOCKED (${cols.blocked.length}):`, ...cols.blocked.map(line), `DONE (${cols.done.length}):`, ...cols.done.map(line)].join("\n");
}
```

```ts
          if (!ev.ok) rejected.push(`'${c.title}' evidence rejected: ${ev.reason}`);

    if (rejected.length > 0) {
      toolResult = `REJECTED:\n- ${rejected.join("\n- ")}\nBoard NOT updated. Either run the missing tool now, or set the card to in_progress and try again next round.`;

      } catch (e: any) {
        toolResult = `manage_tasks failed: ${e?.message ?? String(e)}`;
      }
    }
    if (!toolsUsed.includes("manage_tasks")) toolsUsed.push("manage_tasks");
    messages.push({ role: "tool", tool_call_id: tc.id || "manage_tasks", name: "manage_tasks", content: toolResult });
  }
  reinjectBoard(messages);
}
```

```tsx

const LABELS: Record<TaskStatus,string> = { todo:"To do", in_progress:"Doing", done:"Done", blocked:"Blocked" };
export default function TaskBoard({ board, expanded, onToggle }: { board: TaskBoard | null; expanded: boolean; onToggle: () => void }) {
  if (!board || board.tasks.length === 0) return null;
  const c = { todo:0, doing:0, done:0, blocked:0 };
  for (const t of board.tasks) { if (t.status==='in_progress') c.doing++; else c[t.status as 'todo'|'done'|'blocked']++; }
  if (!expanded) return (
    <button className="board-strip" onClick={onToggle} aria-expanded="false" aria-label={`Task board: ${c.todo} to do, ${c.doing} doing, ${c.done} done${c.blocked?`, ${c.blocked} blocked`:''}`}>
      <span className="board-strip-icon" aria-hidden>▤</span>
      <span className="board-strip-counts">
        {c.todo>0 && <span className="bsc s-todo">{c.todo} to do</span>}
        {c.doing>0 && <span className="bsc s-doing">{c.doing} doing</span>}
        {c.done>0 && <span className="bsc s-done">{c.done} done</span>}
        {c.blocked>0 && <span className="bsc s-blocked">{c.blocked} blocked</span>}
      </span>
      <span className="board-strip-chevron" aria-hidden>▾</span>
    </button>
  );
  return (
    <section className="board" role="region" aria-label="Task board">
      <header className="board-head">
        <span className="board-title">Tasks</span>
        <button className="board-collapse" onClick={onToggle} aria-expanded="true" aria-label="Collapse">▴</button>
      </header>
      <div className="board-cols" role="list">
        {(["todo","in_progress","done","blocked"] as TaskStatus[]).map(s => {
          const cls = s === "in_progress" ? "doing" : s;
          const cards = board.tasks.filter(x => x.status === s);
          return (
            <div key={s} className={`board-col s-${cls}`}>
              <div className="board-col-head"><span className="board-col-label">{LABELS[s]}</span><span className="board-col-count">{cards.length}</span></div>
              <div className="board-col-cards">{cards.map(card => (
                <article key={card.id} className={`board-card s-${cls}`} role="listitem" tabIndex={0}>
                  <div className="board-card-title">{card.title}</div>
                  {s==="done" && card.evidence && <div className="board-card-evidence" title={card.evidence}>✓ {card.evidence}</div>}
                  {s==="blocked" && card.blocker && <div className="board-card-evidence warn">⚠ {card.blocker}</div>}
                  {card.attempt_count > 1 && <span className="board-card-attempt">×{card.attempt_count}</span>}
                </article>
              ))}</div>
            </div>
          );
        })}
      </div>
    </section>
  );
}
```

```rust
// task_board.rs — full type surface

use serde::{Deserialize, Serialize};
use std::path::PathBuf;
use std::sync::Mutex;
use tauri::{AppHandle, Manager};

const CURRENT_SCHEMA_VERSION: u32 = 1;

/// Serializes all task-board file I/O across commands. Tauri already runs
/// command handlers on a worker pool; this Mutex prevents two in-flight
/// saves (agent + UI) from racing on whole-file rewrite.
static FILE_LOCK: Mutex<()> = Mutex::new(());

#[derive(Serialize, Deserialize, Clone, Copy, Debug, PartialEq, Eq, Default)]
#[serde(rename_all = "snake_case")]
pub enum TaskStatus {
    #[default]
    Todo,
    InProgress,
    Done,
    Blocked,
}

#[derive(Serialize, Deserialize, Clone, Debug)]
pub struct Task {
    pub id: String,                          // agent-supplied stable id (e.g. "t_01h...")
    pub title: String,                       // human-readable line item
    pub status: TaskStatus,
    #[serde(default, skip_serializing_if = "Option::is_none")]
    pub evidence: Option<String>,            // required (non-empty) when status == Done
    pub created_at: String,                  // RFC3339, server-assigned
    pub updated_at: String,                  // RFC3339, server-bumped on any field change
    #[serde(default)]
    pub attempt_count: u32,                  // bumped each todo→in_progress transition
}

/// Wire-shape the agent sends to save_task_board. Server fills the rest.
#[derive(Deserialize, Clone, Debug)]
pub struct TaskInput {
    pub id: String,
    pub title: String,
    pub status: TaskStatus,
    #[serde(default)]
    pub evidence: Option<String>,
}

#[derive(Serialize, Deserialize, Clone, Debug, Default)]
pub struct Board {
    pub conversation_id: String,
    pub updated_at: String,
    pub tasks: Vec<Task>,
}

#[derive(Serialize, Deserialize, Debug)]
pub struct TaskBoardStore {
    #[serde(default = "default_version")]
    pub version: u32,
    #[serde(default)]
    pub boards: Vec<Board>,
}

fn default_version() -> u32 { CURRENT_SCHEMA_VERSION }

impl Default for TaskBoardStore {
    fn default() -> Self {
        Self { version: CURRENT_SCHEMA_VERSION, boards: Vec::new() }
    }
}

#[derive(Serialize, Debug)]
pub struct BoardSummary {
    pub conversation_id: String,
    pub updated_at: String,
    pub total: usize,
    pub todo: usize,
    pub in_progress: usize,
    pub done: usize,
    pub blocked: usize,
}

// ---- path + I/O helpers (mirror history.rs) ----

fn store_path(app: &AppHandle) -> Result<PathBuf, String> {
    let dir = app.path().app_config_dir().map_err(|e| e.to_string())?;
    std::fs::create_dir_all(&dir).map_err(|e| e.to_string())?;
    Ok(dir.join("task_boards.json"))
}

fn load_store(app: &AppHandle) -> TaskBoardStore {
    match store_path(app).and_then(|p| std::fs::read_to_string(p).map_err(|e| e.to_string())) {
        Ok(raw) => {
            let mut s: TaskBoardStore = serde_json::from_str(&raw).unwrap_or_default();
            s = migrate_if_needed(s);
            s
        }
        Err(_) => TaskBoardStore::default(),
    }
}

/// Atomic write: serialize → write to <path>.tmp → rename onto target.
fn save_store(app: &AppHandle, store: &TaskBoardStore) -> Result<(), String> {
    let p = store_path(app)?;
    let tmp = p.with_extension("json.tmp");
    let raw = serde_json::to_string_pretty(store).map_err(|e| e.to_string())?;
    std::fs::write(&tmp, raw).map_err(|e| e.to_string())?;
    std::fs::rename(&tmp, &p).map_err(|e| e.to_string())
}

fn migrate_if_needed(store: TaskBoardStore) -> TaskBoardStore {
    // Forward-only migration ladder. Future versions add arms here.
    if store.version > CURRENT_SCHEMA_VERSION {
        // Newer-than-known file (downgrade scenario): leave as-is; we will not
        // overwrite — callers see an empty Vec since fields they expect are absent.
        return store;
    }
    // v1 == current; nothing to do.
    store
}

// ---- Tauri commands ----

#[tauri::command]
pub fn get_task_board(app: AppHandle, conversation_id: String) -> Option<Board> {
    let _g = FILE_LOCK.lock().unwrap();
    load_store(&app)
        .boards
        .into_iter()
        .find(|b| b.conversation_id == conversation_id)
}

#[tauri::command]
pub fn save_task_board(
    app: AppHandle,
    conversation_id: String,
    tasks: Vec<TaskInput>,
) -> Result<Board, String> {
    let _g = FILE_LOCK.lock().unwrap();

    // Evidence guard — Done requires non-empty evidence.
    for t in &tasks {
        if matches!(t.status, TaskStatus::Done)
            && t.evidence.as_deref().map(str::trim).unwrap_or("").is_empty()
        {
            return Err(format!(
                "Task '{}' marked done without evidence. Supply an evidence string \
                 describing the tool call(s) and observed result.",
                t.id
            ));
        }
    }

    let mut store = load_store(&app);
    if store.version > CURRENT_SCHEMA_VERSION {
        return Err(format!(
            "task_boards.json schema version {} is newer than this build ({}). \
             Refusing to overwrite to avoid data loss.",
            store.version, CURRENT_SCHEMA_VERSION
        ));
    }

    let now = chrono::Utc::now().to_rfc3339();
    let existing_idx = store.boards.iter().position(|b| b.conversation_id == conversation_id);
    let existing_tasks: Vec<Task> = existing_idx
        .map(|i| store.boards[i].tasks.clone())
        .unwrap_or_default();

    // Diff: preserve created_at + attempt_count; bump updated_at on change.
    let new_tasks: Vec<Task> = tasks.into_iter().map(|inp| {
        let prior = existing_tasks.iter().find(|t| t.id == inp.id);
        let (created_at, mut attempt_count, prior_status, changed) = match prior {
            Some(p) => (
                p.created_at.clone(),
                p.attempt_count,
                Some(p.status),
                p.title != inp.title || p.status != inp.status || p.evidence != inp.evidence,
            ),
            None => (now.clone(), 0, None, true),
        };
        // Bump attempt_count on todo→in_progress (also covers new-and-in_progress).
        let entering_in_progress = matches!(inp.status, TaskStatus::InProgress)
            && prior_status.map_or(true, |s| !matches!(s, TaskStatus::InProgress));
        if entering_in_progress {
            attempt_count = attempt_count.saturating_add(1);
        }
        let updated_at = if changed { now.clone() } else {
            prior.map(|p| p.updated_at.clone()).unwrap_or_else(|| now.clone())
        };
        Task {
            id: inp.id,
            title: inp.title,
            status: inp.status,
            evidence: inp.evidence,
            created_at,
            updated_at,
            attempt_count,
        }
    }).collect();

    let board = Board {
        conversation_id: conversation_id.clone(),
        updated_at: now,
        tasks: new_tasks,
    };

    match existing_idx {
        Some(i) => store.boards[i] = board.clone(),
        None => store.boards.push(board.clone()),
    }
    store.version = CURRENT_SCHEMA_VERSION;
    save_store(&app, &store)?;
    Ok(board)
}

#[tauri::command]
pub fn list_task_boards(app: AppHandle) -> Vec<BoardSummary> {
    let _g = FILE_LOCK.lock().unwrap();
    let mut store = load_store(&app);
    store.boards.sort_by(|a, b| b.updated_at.cmp(&a.updated_at));
    store.boards.into_iter().map(|b| {
        let mut s = BoardSummary {
            conversation_id: b.conversation_id,
            updated_at: b.updated_at,
            total: b.tasks.len(),
            todo: 0, in_progress: 0, done: 0, blocked: 0,
        };
        for t in &b.tasks {
            match t.status {
                TaskStatus::Todo => s.todo += 1,
                TaskStatus::InProgress => s.in_progress += 1,
                TaskStatus::Done => s.done += 1,
                TaskStatus::Blocked => s.blocked += 1,
            }
        }
        s
    }).collect()
}

#[tauri::command]
pub fn clear_task_board(app: AppHandle, conversation_id: String) -> Result<(), String> {
    let _g = FILE_LOCK.lock().unwrap();
    let mut store = load_store(&app);
    store.boards.retain(|b| b.conversation_id != conversation_id);
    save_store(&app, &store)
}
```

### Pitfalls

- **Whole-file rewrite race: agent's runAgent loop may call save_task_board between rounds while the UI's kanban panel polls get_task_board, and both could simultaneously deserialize → mutate → write, losing one update.** → Single static `FILE_LOCK: Mutex<()>` taken at the top of every command (get/save/list/clear). Lock is released before returning. Reads acquire it too so a save mid-deserialize cannot serve a half-written file (paired with atomic temp-file rename in save_store). Lock contention is irrelevant — kanban writes are human-paced.
- **Partial write corruption — power loss / app crash between fs::write start and fs::write completion would leave a truncated JSON that fails deserialization on next launch, dropping the entire task history.** → Atomic write-via-temp-file: serialize → write to `task_boards.json.tmp` → `fs::rename` onto `task_boards.json`. POSIX rename is atomic on the same filesystem. This is also the gap I'd flag in history.rs which uses fs::write directly — we do NOT inherit that risk here. (Out of scope to fix history.rs in this lens.)
- **Client-supplied timestamps would let the agent rewrite history (e.g. claim a task was created earlier to game attempt_count).** → TaskInput excludes created_at/updated_at/attempt_count fields entirely — they are stripped from the wire format. Server is the sole source of timestamps and attempt counts.
- **Done-without-evidence regression — the model could send `{status:'done'}` to claim completion without doing the tool call (root cause #4 of the original bug).** → save_task_board returns Err if ANY task has status Done with missing/whitespace-only evidence, with a message phrased for the agent to self-correct on the next round. The agent loop surfaces command errors back into the conversation, so the model sees its own malformed call rejected.
- **Schema drift: a future field like `blocked_reason`, `owner`, or `due_at` shipped to one user's machine, then they downgrade — newer JSON on disk vs older deserializer corrupts the file.** → Top-level `version: u32` field. On load: version > CURRENT_SCHEMA_VERSION → load read-only, save_task_board refuses to overwrite with a clear error. Forward-only migration ladder in `migrate_if_needed` handles version < CURRENT_SCHEMA_VERSION.
- **Whole-board replacement semantics tempts the agent into truncating: it might send only the tasks it cares about THIS round and silently lose the rest from prior rounds.** → The system prompt re-inject step (handled in the agent.ts lens, not here) MUST re-read get_task_board and merge before save. This module exposes the read primitive cleanly so the agent has zero excuse to skip it. We could later add an `expected_version` optimistic-concurrency token if we see the bug in the wild — flagged for the agent lens.
- **Unbounded growth: a long-lived user with thousands of conversations would balloon task_boards.json the same way conversations.json grows in history.rs, slowing every load/save.** → list_task_boards already sorts by updated_at; we surface that ordering so the UI can offer a 'archive old boards' control later. clear_task_board deletes outright. No proactive eviction in v1 — same posture as history.rs — but the file is much smaller per-conversation (a board is tens of tasks, not hundreds of messages).
- **Mutex poisoning if a panic occurs while the lock is held (e.g. inside serde_json) would permanently break every subsequent task-board command for the app lifetime.** → Use `.lock().unwrap()` deliberately — a poisoned mutex implies a Rust panic in our code, which is a bug we want to crash-loud on, not silently mask. If we see this in telemetry, switch to `.lock().unwrap_or_else(|e| e.into_inner())` to recover. Documenting the choice rather than hiding it.
- **AppHandle::path().app_config_dir() is fallible (rare, e.g. sandbox misconfigured) — bubbling that as a String error means the agent sees a cryptic message and may retry forever.** → Matches history.rs's exact handling: map_err to String, propagate as Err. The agent.ts lens will need to detect 'cannot resolve app config dir' as terminal and stop the multi-task loop. Flagged as an open question.

## Heuristic constant (src/lib/agent.ts, module scope)

```ts
const MULTI_TASK_HEURISTIC = {
  /** Numbered or bulleted list with >=2 items. */
  list: /(?:^|\n)\s*(?:\d+[.)]\s|[-*]\s).{4,}.*\n\s*(?:\d+[.)]\s|[-*]\s).{4,}/m,
  /** Explicit chaining word. */
  chain: /\b(?:and then|after that|then also|, then |, and then |as well as|in addition|plus also)\b/i,
  /** Multiple distinct imperative verbs in one message. */
  multiImperative: /\b(?:send|email|find|search|pull|retrieve|create|schedule|book|post|generate|draft|summarize|read|open|check|analyze|review)\b[\s\S]{0,80}\b(?:send|email|find|search|pull|retrieve|create|schedule|book|post|generate|draft|summarize|read|open|check|analyze|review)\b/i,
  /** Word-count threshold — long asks are usually multi-task. */
  long: (text: string) => text.trim().split(/\s+/).length > 40,
};

function detectMultiTask(text: string): boolean {
  let score = 0;
  if (MULTI_TASK_HEURISTIC.list.test(text)) score += 2;       // strong
  if (MULTI_TASK_HEURISTIC.chain.test(text)) score += 1;
  if (MULTI_TASK_HEURISTIC.multiImperative.test(text)) score += 1;
  if (MULTI_TASK_HEURISTIC.long(text)) score += 1;
  return score >= 2;
}
```

```ts
const BASE_ROUNDS = 5;
const MAX_ROUNDS_CAP = 30;

export async function runAgent(opts: RunAgentOpts): Promise<RunAgentResult> {
  const { userText, conversationId, onBoardLoad, onBoardUpdate, /* ... */ } = opts;

  // 1. Load existing board (resumes unfinished work cross-session).
  let board: TaskBoard | null = await getTaskBoard(conversationId).catch(() => null);
  onBoardLoad?.(board);

  // 2. Detect multi-task BEFORE round 0.
  const mustDecompose = detectMultiTask(userText) || (board?.cards.some(c => c.status !== "done") ?? false);

  // 3. Build initial messages with optional DECOMPOSE prompt + initial board snapshot.
  const decomposeMsg: ChatMessage | null = mustDecompose ? {
    role: "system",
    content:
      "DECOMPOSE FIRST. This request has multiple tasks (or an unfinished board exists). " +
      "Your FIRST tool call must be manage_tasks with the complete task list. Do not call any " +
      "other tool or answer the user until the board is set. One card in_progress at a time. " +
      "Cards may only be marked 'done' AFTER you have actually called the relevant tool this round.",
  } : null;

  let boardSnapshotIndex = -1;
  const injectBoardSnapshot = (msgs: ChatMessage[]) => {
    if (!board || board.cards.length === 0) return;
    const snap: ChatMessage = {
      role: "system",
      content: `Current board (${board.cards.length} cards, version ${board.version}):\n` +
        board.cards.map(c => `- [${c.status}] ${c.title}${c.evidence ? ` (evidence: ${c.evidence})` : ""}${c.blocker ? ` (blocked: ${c.blocker})` : ""}`).join("\n") +
        `\nRemaining: ${board.cards.filter(c => c.status !== "done").length}. Continue working until all cards are done or blocked.`,
    };
    if (boardSnapshotIndex >= 0) msgs[boardSnapshotIndex] = snap;
    else { msgs.push(snap); boardSnapshotIndex = msgs.length - 1; }
  };

  const messages: ChatMessage[] = [
    { role: "system", content: settings.system_prompt },
    dateMsg,
    ...history,
    ...planMsg,
    ...(decomposeMsg ? [decomposeMsg] : []),
    { role: "user", content: userContent as string },
  ];
  injectBoardSnapshot(messages);

  // 4. Dynamic round budget — grows after manage_tasks lands.
  let maxRounds = BASE_ROUNDS;

  // 5. Track tool calls per round for evidence gating.
  let toolsThisRound = new Set<string>();
  let boardSetThisTurn = false;

  for (let round = 0; round < maxRounds; round++) {
    toolsThisRound = new Set();

    // Force tool_choice on round 0 only if multi-task detected.
    const toolChoice = round === 0 && mustDecompose ? "required" : undefined;
    const res = await llmComplete(/* ..., */ TOOL_DEFS, settings.max_tokens, toolChoice);
    const toolCalls = res.tool_calls ?? [];

    if (toolCalls.length === 0) {
      const content = res.content ?? "";
      // Board-intent narration without a board set → nudge to call manage_tasks.
      if (!boardSetThisTurn && BOARD_INTENT.test(content)) {
        messages.push({
          role: "system",
          content: "You narrated a plan without calling manage_tasks. Call manage_tasks NOW with the complete task list. Do not answer the user yet.",
        });
        continue;
      }
      // Existing ACTION_CLAIM nudge (unchanged behavior).
      if (!nudged && ACTION_CLAIM.test(content)) { /* ... */ continue; }

      // If board has unfinished work, refuse to finalize.
      if (board && board.cards.some(c => c.status === "todo" || c.status === "in_progress")) {
        messages.push({
          role: "system",
          content: `${board.cards.filter(c => c.status !== "done").length} cards remain on the board. Continue working. Do NOT respond to the user until all cards are done or you need clarification.`,
        });
        continue;
      }
      return { content: content.trim(), tools: toolsUsed, route };
    }

    messages.push({ role: "assistant", content: "", tool_calls: toolCalls });

    for (const tc of toolCalls) {
      // ... existing per-tool dispatch ...
      const result = await executeTool(tc.function.name, tc.function.arguments, {
        conversationId,
        toolsThisRound,
        currentBoard: board,
      });

      if (tc.function.name !== "manage_tasks") toolsThisRound.add(tc.function.name);

      messages.push({ role: "tool", tool_call_id: tc.id, name: tc.function.name, content: result });

        if (board) {

          maxRounds = Math.min(MAX_ROUNDS_CAP, BASE_ROUNDS + 2 * board.cards.length);
        }
      }
    }

    // End-of-round: re-inject latest board snapshot.
    injectBoardSnapshot(messages);

    // End-of-round reminder when work remains.
    if (board && board.cards.some(c => c.status === "todo" || c.status === "in_progress")) {
      const remaining = board.cards.filter(c => c.status !== "done").length;
      messages.push({
        role: "system",
        content: `${remaining} card(s) remain. Continue working. Do NOT respond to the user until done==all OR you need clarification.`,
```

## executeTool change for manage_tasks (with evidence gate)

```ts
interface ToolContext {
  conversationId: string;
  toolsThisRound: Set<string>;
  currentBoard: TaskBoard | null;
}

async function executeTool(name: string, argsJson: string, ctx: ToolContext): Promise<string> {
  // ... existing switch ...
  case "manage_tasks": {
    const incoming: Array<Partial<TaskCard>> = Array.isArray(args.cards) ? args.cards : [];

    // Invariant 1: exactly one in_progress (collapse extras to todo).
    let sawInProgress = false;
    const sanitized = incoming.map((c) => {
      const status = (c.status ?? "todo") as TaskStatus;
      if (status === "in_progress") {
        if (sawInProgress) return { ...c, status: "todo" as TaskStatus };
        sawInProgress = true;
      }
      return { ...c, status };
    });

    // Invariant 2: evidence gate for done.
    const rejected: string[] = [];
    for (const c of sanitized) {
      if (c.status === "done") {
        const hadTool = ctx.toolsThisRound.size > 0;
        const ev = (c.evidence ?? "").trim();
        // Was this card already done? If so, allow no-op pass-through.
        const wasDone = ctx.currentBoard?.cards.find(x => x.id === c.id)?.status === "done";
        if (!wasDone && (!ev || !hadTool)) {
          rejected.push(c.title ?? c.id ?? "(unnamed)");
        }
      }
      if (c.status === "blocked" && !(c.blocker ?? "").trim()) {
        rejected.push(`${c.title ?? c.id} (blocked needs 'blocker')`);
      }
    }
    if (rejected.length > 0) {
      return `manage_tasks rejected: ${rejected.join(", ")}. ` +
        `Cards marked 'done' require BOTH an 'evidence' string AND at least one non-board tool call this round. ` +
        `Call the actual tool first, then mark the card done in the NEXT round.`;
    }

    // Persist (Rust assigns missing ids, bumps version, sets timestamps).
    const board = await setTaskBoard(ctx.conversationId, sanitized as TaskCard[]);
    const summary = board.cards.map(c => `[${c.status}] ${c.title}`).join("\n");
    return `Board updated (v${board.version}, ${board.cards.length} cards):\n${summary}`;
  }
}
```

### Pitfalls

- **Forcing tool_choice='required' every round breaks the final-answer return path (model can't ever stop calling tools).** → Force ONLY on round 0 AND only when detectMultiTask returns true. All later rounds use default tool_choice='auto'. Pass `toolChoice` as a new optional param to llmComplete that defaults to undefined (no change for existing callers).
- **Re-injecting the board every round via messages.push() bloats context — after 30 rounds you have 30 board snapshots, each ~500 tokens.** → Track `boardSnapshotIndex` and SPLICE the latest snapshot in place (replace, not append). Old assistant/tool messages still describe what happened, but only ONE current-truth board system message exists at any time.
- **Evidence gate is fooled by a model that calls a no-op tool just to satisfy the check (e.g. brain_search with a junk query, then marks all 5 cards done).** → Evidence string must be non-empty AND the gate runs PER CARD per round, not per board. Combined with `boardSetThisTurn` plus the end-of-round 'cards remain' nudge, the model can mark at most one card done per round (since it needs a fresh tool call). Real abuse would require the model to call a different real tool for each card — at which point work is actually being done.
- **Model emits a manage_tasks call with mangled JSON (Qwen3-8B is known to do this).** → Existing executeTool already wraps `JSON.parse(argsJson)` in try/catch and falls through to `args = {}`. Add an explicit empty-cards check: if `incoming.length === 0` and there's no existing board, return 'manage_tasks called with empty cards array — provide at least one task or do not call this tool'. The model will retry on the next round.
- **Loading the board cross-session resurrects a stale plan from yesterday and the model tries to complete tasks unrelated to the new user message.** → On round 0, if `board` exists AND its `updatedAt` is older than 24h, treat as fresh start (set `board = null` locally and call setTaskBoard with [] to clear in Rust). Threshold lives in one constant `BOARD_TTL_HOURS = 24`.
- **Tool_choice='required' on a model that doesn't support it (some local builds) hard-errors.** → llmComplete already withRetry-wraps calls. Wrap the round-0 required call in a try/catch: if it errors with a 'tool_choice' message, retry once without the constraint and emit a system-message warning that 'manage_tasks is recommended' instead. Log once via console.warn so we know which models are missing the param.
- **Heuristic false-positives — short single-task asks ('book Josh and Sarah a meeting at 3pm') trip the multi-imperative regex and force unnecessary board overhead.** → Score-based gate (>=2 signals required) makes single trips harmless. 'book Josh and Sarah a meeting' hits multiImperative only (book→meeting); needs another signal. Empirically tunable: bump threshold to 3 if false-positives appear in production. Keep the constant exported so it's easy to find.
- **ACTION_CLAIM expansion now matches normal assistant phrasings like 'I'll start by reading the docs' inside a perfectly valid first-tool-call reply.** → The nudge only fires when `toolCalls.length === 0` — if the model both narrates AND calls a tool, ACTION_CLAIM is never even evaluated. So the widened regex can only false-fire on no-tool replies, and the worst case is one extra nudge round which we already cap with `nudged = true`.
- **BOARD_INTENT and ACTION_CLAIM both fire on the same content, causing TWO consecutive nudge messages.** → Check BOARD_INTENT FIRST and `continue;` immediately. ACTION_CLAIM only runs if BOARD_INTENT didn't match. Order matters.
- **onBoardUpdate fires inside the per-tool loop; if the UI re-renders the kanban panel mid-round, the user sees a 'done' card before the actual side-effecting tool has fired (e.g. send_email might still be pending in a later tc within the same toolCalls array).** → manage_tasks must be the LAST tool call in any given assistant turn (encode this in the tool description). In practice Qwen models emit one tool call per turn so this is mostly moot, but the description guidance handles multi-tool-per-turn models.

## Evidence validator

```ts
// Real evidence references something the model couldn't have hallucinated:
// a tool actually called this round, a path, a URL, an ID, or quoted/backticked artifact.
export function validateEvidence(
  evidence: string,
  toolsThisRound: string[]
): { ok: boolean; reason?: string } {
  if (!evidence || evidence.trim().length < 8) {
    return { ok: false, reason: "evidence too short (<8 chars)" };
  }
  const lower = evidence.toLowerCase();

  // (a) names a tool called this round
  const toolNameHit = toolsThisRound
    .filter((t) => t !== "manage_tasks")
    .some((t) => lower.includes(t.toLowerCase()));

  // (b) absolute path
  const PATH = /(?:\/Users\/|\/Library\/|\/Applications\/|~\/)[\w\-./ ]+/;
  // (c) URL
  const URL_RE = /\bhttps?:\/\/[^\s)]+/i;
  // (d) quoted or backticked identifier, or recognizable ID prefix
  const ID_RE = /(?:`[^`]+`|"[^"]{3,}"|\b(?:msg_|event_|conv_|file_|task_|AAMk|AQMk)[A-Za-z0-9_\-]{4,})/;
  // (e) numeric artifact (count, line number, byte size)
  const NUM_RE = /\b\d{2,}\b/;

  if (toolNameHit || PATH.test(evidence) || URL_RE.test(evidence) || ID_RE.test(evidence) || NUM_RE.test(evidence)) {
    return { ok: true };
  }
  return {
    ok: false,
    reason:
      "evidence must reference a tool that ran this round, a file path, a URL, a quoted/backticked ID, or a numeric artifact",
  };
}
```

## Done-transition validator

```ts
export interface ValidationContext {
  toolsThisRound: string[];  // names of tools called THIS round, excluding manage_tasks
  prevCardState?: Card;      // card state before this manage_tasks call
}

export function validateDoneTransition(
  card: Card,
  ctx: ValidationContext
): { ok: boolean; reason?: string } {
  if (card.status !== "done") return { ok: true };

  // Real work must have happened this round (any tool other than manage_tasks).
  const realWork = ctx.toolsThisRound.some((t) => t !== "manage_tasks");
  if (!realWork) {
    return {
      ok: false,
      reason: `card '${card.title}' marked done but no real tool was called this round. Either run the work now or set status back to in_progress.`,
    };
  }

  // Evidence string is required and must look real.
  if (!card.evidence) {
    return { ok: false, reason: `card '${card.title}' marked done without evidence string.` };
  }
  const ev = validateEvidence(card.evidence, ctx.toolsThisRound);
  if (!ev.ok) {
    return {
      ok: false,
      reason: `card '${card.title}' evidence rejected: ${ev.reason}`,
    };
  }
  return { ok: true };
}

export function validateAttemptCeiling(card: Card): { ok: boolean; reason?: string } {
  if (card.status === "done" || card.status === "blocked") return { ok: true };
  if (card.attempt_count >= 3) {
    return {
      ok: false,
      reason: `card '${card.title}' has attempt_count=${card.attempt_count}. Set status to "blocked" with blocker_reason explaining what is needed, or provide concrete evidence and mark done.`,
    };
  }
  return { ok: true };
}
```

```ts
  if (!msg) return 0;
  // Numbered list?
  const numberedMatches = msg.match(/(?:^|\n)\s*\d+[.)]\s+\S/g);
  if (numberedMatches && numberedMatches.length >= 2) return numberedMatches.length;

  // Count distinct imperative verbs near start of sentences.
  const lower = msg.toLowerCase();
  const verbHits = new Set<string>();
  for (const v of IMPERATIVE_VERBS) {
    const re = new RegExp(`(?:^|[.!?]\\s+|\\sand\\s+|\\sthen\\s+|,\\s+)${v}\\b`, "i");
    if (re.test(lower)) verbHits.add(v);
  }
  if (verbHits.size >= 2) return verbHits.size;

  // Connector phrases like "X and then Y"
  const connectorParts = msg.split(CONNECTORS).filter((s) => s.trim().length > 4);
  if (connectorParts.length >= 2) return connectorParts.length;

  return 1;
}

export function isMultiTask(userMsg: string): boolean {
  return estimateTaskCount(userMsg) >= 2;
}
```

## Stale-board detector

```ts
function tokenize(s: string): Set<string> {
  return new Set(
    s.toLowerCase()
      .replace(/[^\w\s]/g, " ")
      .split(/\s+/)
      .filter((w) => w.length >= 4) // drop stopwords cheaply
  );
}
export function jaccard(a: Set<string>, b: Set<string>): number {
  if (a.size === 0 || b.size === 0) return 0;
  let inter = 0;
  for (const x of a) if (b.has(x)) inter++;
  return inter / (a.size + b.size - inter);
}
export function isStaleBoard(userMsg: string, board: KanbanBoard): boolean {
  const live = board.cards.filter((c) => c.status !== "done" && c.status !== "blocked");
  if (live.length === 0) return false;
  if (userMsg.trim().length < 20) return false;
  const userTok = tokenize(userMsg);
  const boardTok = tokenize(
    live.map((c) => `${c.title} ${c.description ?? ""}`).join(" ")
  );
  return jaccard(userTok, boardTok) < 0.15;
}
```

```ts
    role: "system",
    content: `Detected ${taskCount} tasks. Round 0: call manage_tasks to create one card per task BEFORE doing any work. Set the first card to in_progress.`,
  });
}

  // Re-inject current board state at the top of every round (after round 0 it has cards).
  if (board.cards.length > 0) {

    });

  if (toolCalls.length === 0) {

      nudged = true;

        content: `NO TOOL CALLED. Detected narration pattern: ${pattern}. If real work is needed, call manage_tasks (to plan) or a domain tool (web/file/email/calendar/brain) NOW. Describing the plan in prose is not execution.`,
      });

    onToken?.(answer);
    await saveKanban(conversationId, board);
    return { content: answer, tools: toolsUsed, route };

  messages.push({ role: "assistant", content: "", tool_calls: toolCalls });

  const roundToolNames: string[] = [];
  // Defer manage_tasks calls so we know what other tools fired this round.
  const pendingManageTasks: { tc: any; aobj: any }[] = [];

  for (const tc of toolCalls) {
    if (signal?.aborted) throw new Error("aborted");
    let aobj: any = {};
    try { aobj = tc.function.arguments ? JSON.parse(tc.function.arguments) : {}; } catch {}
    if (tc.function.name === "manage_tasks") {
      pendingManageTasks.push({ tc, aobj });
      continue;

      tool_call_id: tc.id || tc.function.name,
      name: tc.function.name,
      content: result,
    });

      const prev = board.cards.find((c) => c.id === proposed.id);

      // Bump attempt_count on transition into in_progress.
      if (proposed.status === "in_progress" && prev?.status !== "in_progress") {
        proposed.attempt_count = (prev?.attempt_count ?? 0) + 1;
      } else {
        proposed.attempt_count = prev?.attempt_count ?? proposed.attempt_count ?? 0;

      const doneCheck = validateDoneTransition(proposed, {
        toolsThisRound: roundToolNames,
        prevCardState: prev,
      });

      const ceilingCheck = validateAttemptCeiling(proposed);

          if (c.id !== proposed.id && c.status === "in_progress") c.status = "todo";

      if (idx >= 0) board.cards[idx] = { ...board.cards[idx], ...proposed, updated_at: Date.now() };
      else board.cards.push({ ...proposed, created_at: Date.now(), updated_at: Date.now() });
    }

    if (!toolsUsed.includes("manage_tasks")) toolsUsed.push("manage_tasks");
    const resultStr = rejections.length

      : `OK. Board:\n${renderBoard(board)}`;
    messages.push({
      role: "tool",
      tool_call_id: tc.id || "manage_tasks",
      name: "manage_tasks",
      content: resultStr,
    });
  }

  await saveKanban(conversationId, board);
  onState?.("thinking");
}

// Exhausted rounds — final answer without tools.
const final = await llmComplete(endpoint.baseUrl, endpoint.token, endpoint.model, messages, undefined, settings.max_tokens);
const answer = (final.content ?? "").trim();
onToken?.(answer);
return { content: answer, tools: toolsUsed, route };
```

## renderBoard helper

```ts
function renderBoard(b: KanbanBoard): string {
  const cols: Record<CardStatus, Card[]> = { todo: [], in_progress: [], done: [], blocked: [] };
  for (const c of b.cards) cols[c.status].push(c);
  const line = (c: Card) =>
    `  - [${c.id.slice(0, 6)}] ${c.title}${c.attempt_count ? ` (attempts=${c.attempt_count})` : ""}${c.evidence ? ` ev: ${c.evidence.slice(0, 60)}` : ""}${c.blocker_reason ? ` blocker: ${c.blocker_reason}` : ""}`;
  return [
    `IN_PROGRESS (${cols.in_progress.length}):`,
    ...cols.in_progress.map(line),
    `TODO (${cols.todo.length}):`,
    ...cols.todo.map(line),
    `BLOCKED (${cols.blocked.length}):`,
    ...cols.blocked.map(line),
    `DONE (${cols.done.length}):`,
    ...cols.done.map(line),
  ].join("\n");
}
```

### Pitfalls

- **Reasoning models (gemma-4-26b-a4b) emit harmony tokens like '<|channel|>thought|>' in content. If we re-inject board state into content fields the model has produced, LM Studio's template parser crashes with 'Failed to parse input at pos 0' (already documented at agent.ts line 953).** → Only inject board state via fresh role:'system' messages we author, never echo back the model's content. The existing pattern of pushing { role:'assistant', content:'', tool_calls: toolCalls } at line 956 must be preserved verbatim.
- **Forcing tool_choice on round 0 with a model that already wants to call a different tool can cause LM Studio to return an empty content/no tool_calls response (observed with qwen3-8b when tool_choice conflicts with the model's own routing).** → Only force tool_choice when isMultiTask returns true AND the board is empty. Single-task requests get the model's natural choice. Add a one-shot fallback: if round 0 returns empty with forced tool_choice, retry once with tool_choice='auto'.
- **The NUMBERED_PLAN regex can false-positive on legitimate answers that list facts (e.g. 'Here are the three quotes: 1. ... 2. ...'). If it triggers and we send the nudge, the model may re-emit the same content and we lose a round.** → Only check NUMBERED_PLAN when toolCalls.length === 0 AND content length > 80 chars AND no question mark in the last 100 chars (questions usually mean the model is asking for confirmation, not planning). Also gate with the !nudged latch so it fires at most once.
- **Jaccard tokenization with word-length >= 4 drops short but meaningful identifiers like 'Josh', 'CSS', 'M365'. A user pivoting from 'find Josh's email' to 'pull M365 calendar' would look semantically identical (both have no >=4 char tokens in common with prior board).** → Lower the floor to 3 chars AND add a guard: don't archive if ANY proper-noun-shaped token (capitalized in either source) overlaps. Also require board age > 60s before archive eligibility, so a user clarifying within seconds doesn't lose the board.
- **Evidence validator's numeric-artifact rule (\b\d{2,}\b) accepts hallucinated numbers. The model could write 'Sent 42 emails' as evidence with no real send.** → Require numeric artifact to co-occur with a unit or context token: pair it with a regex like /\d{2,}\s*(?:bytes|chars|lines|results|matches|rows|kb|mb|messages|emails|events|files|tokens|ms)\b/i. Pure numbers alone don't pass; numbers with a recognized unit do. This stops fabricated counts while preserving real tool-output references like '47 results'.
- **If manage_tasks is rejected and the model retries with the SAME rejected payload, we burn rounds in a no-op loop.** → Hash the rejected proposal (JSON.stringify) and track in a per-conversation Set. If the same rejected payload arrives again, escalate the rejection message to include the EXACT prior rejection so the model sees its own loop, AND skip the next round's manage_tasks-only call by injecting a system message forcing a domain tool.
- **attempt_count increments on every transition to in_progress, including legitimate re-pickup after a blocking question to the user. A card that takes 4 user-clarification rounds will hit the ceiling artificially.** → Only increment when the previous status was todo or done. Transitions from blocked -> in_progress (user unblocked it) reset attempt_count to 1. Also expose a manage_tasks action 'reset_attempts' the model can use after a user clarification.
- **Cross-session board persistence means a stale board from yesterday is silently loaded when the user opens the avatar. They could ask an unrelated question and the model sees old cards.** → isStaleBoard runs on EVERY user message (not just round 0 of a new conversation). Also auto-archive any board whose newest card updated_at is > 6 hours old before the new turn, with a system note so the model knows.
- **MAX_ROUNDS = 2 * taskCount + 2 can balloon for adversarial inputs ('do A and B and C and D and E and F and G' -> 16 rounds, each ~60-120s -> 16-32 minute hang).** → Cap maxRounds at 16 absolute. Also enforce a wall-clock budget: if cumulative round time exceeds 8 minutes, break the loop with the standard 'exhausted rounds' final answer path.
- **The narration regex array is checked with .some() which short-circuits, but if two patterns match the same content we lose telemetry on which is firing more.** → Return the first matched name AND log it via a debug counter (in-memory Map<string, number>) flushed to console every N invocations. Lets us tune patterns based on real false-positive rates without changing the gating behavior.
- **Forced manage_tasks on round 0 means the round 0 tool_result will contain the rendered board. On round 1, the model sees both the system 'CURRENT BOARD' message AND the tool_result 'OK. Board: ...' — duplicate context wastes tokens and can confuse small models.** → On rounds where the previous round's last tool_result was manage_tasks, skip the system-message board re-injection. Track lastRoundEndedWithManageTasks: boolean and gate the top-of-loop injection on !lastRoundEndedWithManageTasks.
